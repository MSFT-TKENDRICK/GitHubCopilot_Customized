# Fulfillment dashboard release notes

This update adds a carrier-status filter and clearer empty-state wording for delayed orders.

## Notes for support

- Use the new filter before you reopen a ticket.
- The dashboard now groups orders by the most recent carrier event.
- The escalation reminder appears only when the ticket has been idle for more than 48 hours.

If you spot a case that still read awkwardly, leave a short note so the documentation can be updated.
